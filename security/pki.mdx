**PKI** creates trusted digital identities via **CA hierarchies** issuing **X.509 certificates** binding public keys to entities (users/devices/domains), enabling secure **asymmetric crypto** (TLS, signatures, encryption)—Root CA (offline) → Intermediate CA → End-entity certs with CRL/OCSP revocation.[keyfactor] Senior architects deploy enterprise PKI for mTLS zero-trust, code signing, IoT device auth, trading CA management complexity for scalable trust vs self-signed cert chaos.

## Interview Summary
**Hierarchy**: Root CA (self-signed, offline vault) → Subordinate CAs (online issuance) → End-entity certs (1yr expiry); **CSR Flow**: Entity generates keypair → CSR (public key + attrs) → CA signs → cert chain validation; **Revocation**: CRL (lists) + OCSP (real-time); **Standards**: X.509 v3, TLS 1.3.[keyfactor][fortinet]

## Keywords & Patterns
- **Components**: Root CA, Intermediate CA, End-entity cert, CRL, OCSP responder.[keyfactor]
- **CSR**: Certificate Signing Request (public key + DN + SANs).[keyfactor]
- **Chain**: `End-entity → Intermediate → Root` (trust anchor).[fortinet]
- **Revocation**: CRL (periodic lists), OCSP (stapling/real-time).[keyfactor]
- **Extensions**: SANs (DNS/IP/email), Key Usage (signing/encryption).[fortinet]
- **Stores**: Java KeyStore (JKS), PKCS#12 (P12), Windows Certificate Store.[keyfactor]

## Common Trade-offs + Questions
PKI: Scalable trust vs CA ops overhead; CRL: Simple vs OCSP latency; Root offline: Secure vs issuance delays; SANs: Flexible vs cert size; Short expiry: Secure vs renewal burden.[keyfactor]

**Example questions**:
- Root CA compromise? (Publish CRL, generate new root).[keyfactor]
- CRL vs OCSP? (Batch vs real-time, stapling mitigates).[fortinet]
- mTLS workflow? (Client cert auth + server cert validation).[keyfactor]

## Cheat-Sheet Q&A
**Q: PKI hierarchy?** A: Root (offline) → Intermediate → End-entity.[keyfactor]  
**Q: CSR contents?** A: Public key + DN + SANs + signature (prove possession).[keyfactor]  
**Q: CRL vs OCSP?** A: CRL=lists (cached), OCSP=real-time (stapled).[fortinet]  
**Q: Root compromise?** A: New root + CRL publication (no revocation).[keyfactor]  
**Q: SANs purpose?** A: Multiple DNS/IP/email in one cert.[keyfactor]

## Data Structures & Algorithms
