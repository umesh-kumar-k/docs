### **8. Declarative Pipeline **[8]
**Full Syntax Reference**:
```groovy

  agent { docker 'maven:3.8' }  // Container agent
  environment { VERSION = '1.0' }
  options { timeout(time: 1, unit: 'HOURS') }
  stages {
    stage('Parallel Tests') {
      parallel {
        stage('Unit') { steps { sh 'mvn test' } }
        stage('Lint') { steps { sh 'eslint .' } }
      }
    }
  }
  post {
    always { junit '**/*.xml' }
    success { slackSend 'Build passed!' }
  }
}
```

## **ðŸŽ¯ Interview Cheat Sheet (Q&A)**

**Q: Jenkins architecture?**  
A: **Controller** orchestrates **Agents** (Permanent/Ephemeral). **Nodes** = either. **Jobs** execute on agents.[2]

**Q: Declarative pipeline structure?**  
A: `pipeline { agent stages post } stage { steps {} }`[8]

**Q: Cron `H/15 * * * *` meaning?**  
A: **Hash/15 minutes** - distributes load across 15-min windows[5]

**Q: Poll SCM vs Webhook?**  
A: **Poll**: Jenkinsâ†’Git (load-heavy); **Webhook**: Gitâ†’Jenkins (instant)

**Q: Freestyle vs Pipeline pros/cons?**  
A: **Freestyle**: Easy UI, no versioning; **Pipeline**: Git-controlled, visual stages[4]

**Q: Ephemeral Agent advantages?**  
A: **Clean slate** (no leftovers), **cost-effective** (spot/cloud), **scalable**[2]

**Q: Top 3 plugins for Java CI/CD?**  
A: **Git, Maven Integration, Pipeline** + **SonarQube, Docker, Kubernetes**[9]
