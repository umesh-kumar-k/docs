## Detailed Key Topics
**Step-by-Step Framework**
1. **Understand Requirements**
   - Functional: "Users do X" (search, book, stream).
   - Non-functional: Scale (DAU/QPS), latency, consistency.

2. **High-Level Architecture**
```
Client → Load Balancer → App Servers (stateless)
                             ↓
                       Cache Layer (Redis)
                             ↓
                       Database Layer
```

3. **Identify Bottlenecks**
   - **Traffic**: High RPS → LB + caching.
   - **Storage**: Big data → sharding/replication.
   - **Compute**: Heavy tasks → async queues.

4. **Scale Strategies**
```
Vertical: ↑CPU/RAM (simple, hardware limits)
Horizontal: Add servers + LB (complex, infinite scale)
```

**Core Components Breakdown**
| Component | Purpose | Tech Examples |
|-----------|---------|---------------|
| **Load Balancer** | Distribute traffic | Nginx, HAProxy |
| **Cache** | Hot data (95% hit) | Redis, Memcached |
| **Database** | Persistence | Postgres, DynamoDB |
| **Queue** | Async tasks | Kafka, SQS |
| **Search** | Full-text | Elasticsearch |

## Design Patterns
```
1. Cache-Aside: App checks cache → DB miss → populate
2. Read Replicas: Scale reads (master for writes)
3. Sharding: Split data (userId % N)
4. CQRS: Separate read/write models
```

## Best Practices (Detailed)
**Scaling Checklist**
```
High Reads: Cache + Read Replicas + CDN
High Writes: Sharding + Write Queues
High Availability: Multi-AZ + Auto-failover
Low Latency: Edge cache + Regional DBs
```

**Front-End Considerations**
```
CDN (static assets)
Service Workers (offline)
SSR (SEO + perf)
Http2 + Compression
Lazy loading + Code splitting
```

## Interview Cheat Sheet (Q&A)
**Q: "1M concurrent users?"**  
A: "LB → 1000 app servers (1K RPS each) + Redis (95% cache hit) → 50 DB shards."[1]

**Q: "Database choice?"**  
A: "SQL: ACID/relations (bookings). NoSQL: Scale/flex (feeds). Hybrid."[1]

**Q: "Cache strategy?"**  
A: "Cache-aside: App → Redis → DB. TTL=1hr. Write-through critical data."[1]

**Q: "Single point failure?"**  
A: "LB failover + Redis cluster + DB master-slave promotion."[1]

## Advanced Topics
