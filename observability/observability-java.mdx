Java observability via **OpenTelemetry (OTel)** + **Grafana Cloud** + **Digma IDE plugin** creates developer-centric pipeline: OTel Java agent auto-instruments Spring Boot → Grafana Alloy Collector (pipeline as code) → dual export (Grafana traces + Digma code analysis), shortening feedback loop from deployment to IDE insights.[grafana] Senior architects deploy OTel agent (`-javaagent:opentelemetry-javaagent.jar`) + Alloy for vendor-neutral MELT, trading minimal code changes for runtime code insights (hot paths, dead code, spans).

## Interview Summary
**Pipeline**: Java app → OTel agent (auto-tracing) → Alloy Collector (`config.river`) → Grafana Cloud (traces) + Digma (IDE insights); **Developer Flow**: Run/debug → Digma analyzes traces → IDE CodeLens (usage/dead code) + insights panel; **Alloy**: River config receives OTLP (4317/4318), batches, exports dual destinations.[grafana]
