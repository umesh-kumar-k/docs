**Monitoring** reacts to known metrics/alerts (what happened?), **Observability** infers unknown system states via MELT analysis (why/how?), enabling proactive root cause via correlation/AIOps—monitoring for simple systems, observability for distributed complexity (microservices).[newrelic] Senior architects deploy observability (OTel+unified platforms) for MTTR < 1 5min, trading data volume/cost for predictive failure detection vs monitoring's reactive dashboards.

## Interview Summary
**Monitoring**: Metrics/thresholds/logs → alerts (reactive); **Observability**: MELT (Metrics/Events/Logs/Traces) → AI correlation → actionable insights (proactive); monitoring subset of observability data; modern distributed systems demand observability (unknown unknowns).[newrelic] **DevOps/SRE**: Monitoring for quick response, observability for root cause prevention.
