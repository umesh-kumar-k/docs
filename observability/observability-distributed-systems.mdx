Distributed systems observability requires **context propagation** (TraceID/SpanID via HTTP headers/gRPC metadata) + **MELT instrumentation** across service boundaries using **OTel Java agent** (auto) or **manual spans** (`tracer.spanBuilder`), visualized via **Jaeger/Zipkin** (traces) + **Prometheus/Grafana** (metrics/logs) for service maps/root cause.[aelkz][baeldung][geeks] Senior architects deploy **service mesh** (Istio/Linkerd) or **OTel Collector** for uniform telemetry, trading overhead for end-to-end visibility in 100+ microservices.

## Interview Summary
**Stack**: OTel agent (`-javaagent`) → Collector (OTLP) → Jaeger (traces) + Prometheus (metrics) + Loki (logs) + Grafana (dashboards); **Propagation**: W3C TraceContext (`traceparent/tracesate`); **Patterns**: `@WithSpan`, `Context.current()`; **GitHub Demo**: Spring Boot + Camel + Vert.x + OpenShift (Prometheus/Grafana/Jaeger).[aelkz][baeldung]

## Keywords & Patterns
- **Instrumentation**: OTel Java agent (auto), `io.opentelemetry.api.Tracer`, `@WithSpan`.[baeldung]
- **Propagation**: `traceparent:00-<traceId>-<spanId>-01`, `tracestate`, Baggage.[geeks]
- **Spans**: Parent-child trees, `span.setAttribute("user.id", 123)`.[aelkz]
- **Service Mesh**: Istio Envoy (auto sidecar tracing).[baeldung]
- **Collector**: OTel gateway (batch/sample/export OTLP).[geeks]

## Common Trade-offs + Questions
OTel Agent: Zero-code/consistent vs perf overhead (5-10%); Manual spans: Granular vs maintenance; Jaeger: Rich UI vs Zipkin lightweight; Service mesh: Auto-tracing vs resource overhead; Full MELT: Complete vs cost explosion.[baeldung][geeks]

**Example questions**:
- Trace propagation? (HTTP headers: `traceparent/tracestate`).[baeldung]
- OTel vs Zipkin? (Vendor-neutral vs legacy).[geeks]
- High traffic sampling? (Head/tail-based).[aelkz]

| Approach | Overhead | Consistency | Maintenance |
|----------|----------|-------------|-------------|
| OTel Agent | 5-10% | Auto | Zero [baeldung] |
| Manual Spans | Low | Custom | High [geeks] |
| Service Mesh | 10-20% | Auto | Infra [baeldung] |

## Data Structures & Algorithms
**Trace Trees**: Span DAG (parent-child), flame graphs. **Sampling**: Probabilistic (ratio), tail-based (post-collect). **Correlation**: TraceID joins (logs/metrics). **Service Maps**: Graph algorithms (dependencies). **Anomaly**: Isolation Forest (latency).[baeldung][geeks]
