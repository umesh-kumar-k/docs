Spring Boot 3 observability auto-configures **Micrometer Observation API** + **OTel/Brave tracing** via `spring-boot-starter-actuator`, enabling MELT (metrics/logs/traces) with `@Observed` annotation, `ObservationRegistry`, and handlers for HTTP/services—zero-config for web endpoints, custom via `Observation.createNotStarted()`.[baeldung] Senior architects leverage for microservices tracing (Zipkin/OTel), testing via `TestObservationRegistry/SimpleTracer`, trading minimal code for vendor-neutral instrumentation.

## Interview Summary
**Core**: Micrometer Observation API (`ObservationRegistry` + handlers) auto-instruments HTTP/services; **Actuator** registers metrics (`/actuator/metrics`), traces (Zipkin/OTel); **Tracing**: `micrometer-tracing-bridge-otel/brave` + `@Observed` aspect; **Testing**: `TestObservationRegistry` + AssertJ assertions.[baeldung] **Lifecycle**: `start() → openScope() → error() → stop()` or `observe(() → {})`.

## Keywords & Patterns
- **Annotations**: `@Observed(name="user.service")` (AOP auto).[baeldung]
- **API**: `Observation.createNotStarted("name", registry).observe(() → {})`.[baeldung]
- **Registry**: `ObservationRegistry` (injectable, auto-configured).[baeldung]
- **Handlers**: `ObservationHandler<Context>`, `DefaultMeterObservationHandler`, `ObservationTextPublisher`.[baeldung]
- **Tracing**: `micrometer-tracing-bridge-otel`, `SimpleTracer` (tests).[baeldung]
- **Endpoints**: `/actuator/metrics/{name}`, `/actuator/prometheus`.[baeldung]

## Common Trade-offs + Questions
Spring Observability: Zero-config/standards vs custom agent control; Micrometer: Vendor-neutral vs native SDKs; `@Observed`: Simple vs manual `Observation` control; OTel: Future-proof vs Brave maturity.[baeldung]

**Example questions**:
- `@Observed` vs manual? (Annotation simple, manual low-level).[baeldung]
- Test traces? (`TestObservationRegistry` + `SimpleTracer`).[baeldung]
- HTTP auto-tracing? (Actuator `ServerHttpObservationFilter`).[baeldung]

## Data Structures & Algorithms
**Observation**: Span tree (parent-child), context propagation (W3C TraceContext). **Metrics**: Timer (histogram), Counter (monotonic). **Registry**: ConcurrentHashMap (handler lookup). **Sampling**: Probabilistic/reservoir. **Testing**: In-memory span store + AssertJ.[baeldung]
