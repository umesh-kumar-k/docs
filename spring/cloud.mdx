Spring Cloud extends Spring Boot for building scalable, resilient cloud-native microservices with patterns like service discovery and configuration management.[1][2]

## Advanced Concepts
Spring Cloud tackles distributed system challenges through centralized configuration via Config Server, dynamic service discovery with Eureka or Consul, and client-side load balancing using Spring Cloud LoadBalancer (replacing Ribbon).  Circuit breakers with Resilience4j prevent cascading failures, while API Gateway handles routing and security; distributed tracing via Sleuth integrates with OpenTelemetry for observability.[2][3][4][1]

## Key Classes, Interfaces, Configurations
Core elements include `@EnableConfigServer` on main class for Config Server with `spring.cloud.config.server.git.uri` in `application.yml`; `@EnableEurekaServer` for discovery server.  Clients use `@LoadBalancerClient` with `ServiceInstanceSupplier` bean, `@FeignClient` interface for declarative HTTP, and `bootstrap.properties` with `spring.cloud.config.uri`.  Resilience4j configs like `resilience4j.circuitbreaker.instances` manage timeouts and fallbacks.[3][4][1]

## Interview Summary
Master Spring Cloud for senior roles by explaining how it simplifies microservices: Eureka registers services for dynamic discovery, Gateway routes traffic with filters, Config Server externalizes props via Git.  Highlight resilience (circuit breakers over Hystrix), observability (Sleuth + Zipkin), and tradeoffs like Eureka's operational overhead vs Kubernetes native discovery.  Demonstrate via starters like `spring-cloud-starter-netflix-eureka-client`.[4][1][2]

## Keywords & Patterns
- **Discovery**: EurekaServer, @EnableDiscoveryClient, ServiceRegistry
- **Config**: ConfigServer, @EnableConfigServer, EnvironmentRepository
- **Load Balancing**: SpringCloudLoadBalancer, @LoadBalanced WebClient, ReactorLoadBalancerExchangeFilterFunction
- **Gateway/Routing**: SpringCloudGateway, RouteLocator, Predicate/SpringCloudGatewayFilterFactory
- **Resilience**: Resilience4jCircuitBreaker, @CircuitBreaker, Bulkhead, Retry
- **Tracing**: Sleuth, Zipkin, OpenTelemetry, BaggagePropagation
- **Messaging**: SpringCloudStream, Kafka/Rabbit binders
- **Patterns**: Client-Side LB, API Proxy, 12-Factor App, Fault Tolerance[2][4]

## Trade-offs & Questions
Eureka adds registry complexity but enables health-aware routing; prefer Spring LoadBalancer over Ribbon for modernity, though it lacks some Ribbon plugins.  Config Server boosts env management but introduces single failure pointâ€”mitigate with replication.[5][3]

- How to replace Ribbon with LoadBalancer in Eureka? (Use `@LoadBalancerClient` + supplier bean)[3]
- Circuit breaker vs retries? (Breakers for latency, retries for transient faults with backoff)[4]
- Gateway vs direct Feign? (Gateway centralizes security/filtering)[6]

## Use Cases & Tools
Netflix uses Eureka/Zuul/Hystrix for massive scale video streaming, improving resilience via circuit breakers.  Tools include start.spring.io for projects, Maven/Gradle BOM (`spring-cloud-dependencies`), AWS Elastic Beanstalk for deployment, Redis/Ehcache for caching, Kafka for events.[7][8][1][6]

Big tech refs: Netflix OSS blogs on Eureka scaling (netflixtechblog.com), Callista on Hystrix integration for microservices fault tolerance.[9][6]

## Cheat-Sheet Q&A
**Q: Core BOM?** A: `<spring-cloud-dependencies>2023.0.x</spring-cloud-dependencies>`[1]
**Q: Enable client discovery?** A: `@EnableDiscoveryClient` + `spring-cloud-starter-netflix-eureka-client`[8]
**Q: Config refresh?** A: `@RefreshScope` + Actuator `/actuator/refresh` POST[5]
**Q: Gateway route?** A: `RouteLocator` bean with `Path` predicate[10]
**Q: Fallback method?** A: `@Backup` in Resilience4j or Hystrix `@Fallback`[4]
