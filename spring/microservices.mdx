Spring Boot modules enable building loosely coupled microservices through auto-configuration, embedded servers, and starters for REST, JPA, and discovery integration.[1][2]

## Advanced Concepts
Spring microservices leverage Spring Cloud for service discovery (Eureka), config management, and resilience patterns like circuit breakers.  Advanced topics include reactive programming with WebFlux for non-blocking I/O, event-driven architecture via Spring Cloud Stream with Kafka/RabbitMQ binders, and containerization with Docker/Kubernetes for orchestration.[2][1]

## Key Classes, Interfaces, Configurations
Controllers use `@RestController` with `@GetMapping` for endpoints; services autowire `@Repository` interfaces extending `JpaRepository<Entity, ID>`.  Configurations include `@SpringBootApplication`, `@EnableDiscoveryClient` for Eureka, `application.yml` with `spring.datasource.url` and `eureka.client.serviceUrl.defaultZone`.  ModelMapper `@Bean` handles DTO mapping; `@Configuration` defines custom beans like services.[1][2]

## Interview Summary
Spring Boot simplifies microservices via starters (`spring-boot-starter-web`, `-data-jpa`) and actuators for health checks, enabling 12-factor apps with externalized config.  Senior architects discuss scaling via Kubernetes, observability with Micrometer/Prometheus, and inter-service communication (REST/gRPC/events). Know how `@EnableAutoConfiguration` reduces boilerplate while `@EntityScan`/`@EnableJpaRepositories` scans for domain models.[2][1]

## Cheat-Sheet Q&A
**Q: Enable JPA repos?** A: `@EnableJpaRepositories(basePackages="...")`[2]
**Q: Multi-profile config?** A: `spring.profiles.active=dev` in `application-dev.yml`[1]
**Q: Health endpoint?** A: Add `spring-boot-starter-actuator`; hit `/actuator/health`[1]
**Q: DTO mapping?** A: `@Bean public ModelMapper modelMapper() { return new ModelMapper(); }`[2]
**Q: Service discovery URL?** A: `eureka.client.serviceUrl.defaultZone=http://localhost:8761/eureka/`[1]
