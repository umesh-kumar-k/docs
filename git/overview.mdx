### **1. Git SSH Keys **[1]
**SSH Key Concept**:
- **Public key** = "lock" (shared)
- **Private key** = "key" (secret)
- **RSA/DSA**: 4096-bit recommended

**Mac/Linux Generation**:
```
ssh-keygen -t rsa -b 4096 -C "email@domain.com"
# Default: ~/.ssh/id_rsa (private), id_rsa.pub (public)
# Passphrase: Optional extra security
eval "$(ssh-agent -s)"  # Start agent
ssh-add ~/.ssh/id_rsa   # Load private key
```

**Windows**: Git Bash or WSL (same commands)

### **2. Git Archive **[2]
**Purpose**: Export **commit/branch/tree** â†’ **tar/zip/tar.gz** (no Git metadata)

**Basic Usage**:
```
git archive --format=tar HEAD > repo.tar           # Current HEAD
git archive --output=repo.tar.gz HEAD ./build/     # Subdir only
git archive --remote=git@github.com:user/repo.git HEAD | tar -x  # Remote
```

**Options**:
| Flag | Effect |
|------|--------|
| `--prefix=app/` | Prepend path |
| `--remote=URL` | Archive from remote |
| `tar.umask` | File permissions |

### **4. Setting Up Repository **[4]
**git init** (Local):
```

git init                    # Creates .git/
git config --global user.name "John"

```

**git clone** (Remote):
```

```

**First Commit**:
```

```

### **7. Undoing Changes **[7]
**Local (Unpublished)**: `git reset`
```
git reset --soft HEAD~1     # Undo commit, keep staging
git reset --mixed HEAD~1    # Undo commit+staging, keep working dir
git reset --hard HEAD~1     # Nuke everything to previous commit
```

**Public (Shared)**: `git revert`
```
git revert HEAD             # New commit undoing last commit
```

**Files Only**:
```
git checkout HEAD -- file.txt  # Restore single file
git clean -fd                  # Remove untracked
```

**Safety**: `git reflog` (90-day history)

### **8. Rewriting History **[8]
**git commit --amend** (Last Commit):
```

git add forgot.txt; git commit --amend --no-edit # Add to last commit

```

**git rebase -i** (Interactive):
```

# Editor opens â†’ combine messages
```

**Squash**: `s` merges commits (clean history)

### **9. Syncing (Remotes) **[9]
**git remote**:
```

**URLs**:

**Fetch/Pull/Push**:

```

### **11. Using Branches **
**Branches = Lightweight Pointers**:
```
git branch feature           # Create (no switch)
git checkout -b feature      # Create+switch
git checkout main            # Switch
git branch -d feature        # Safe delete
git branch -D feature        # Force delete
git branch -a                # All (local+remote)
```

**Remote Branches**:
```
git push origin feature
git push origin :feature     # Delete remote
```

### **12. Comparing Workflows **
| Workflow | Branching | Scale | Use Case |

| **Centralized** | main only | Small | SVNâ†’Git migration |

**Guidelines**:
