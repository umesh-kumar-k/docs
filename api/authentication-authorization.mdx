API authentication verifies identity (who?), authorization checks permissions (what?), using methods from Basic Auth (simplest, least secure) to mTLS (mutual certs, highest security).[zuplo][apideck] Senior architects select based on threat model: API keys for devx/public APIs, JWT for stateless microservices, OAuth2 for 3rd-party delegation, mTLS for zero-trust internal.[kong][frontegg]

## Interview Summary
Core methods: **Basic Auth** (Base64 username:pass, HTTPS-only), **API Keys** (simple strings, scoped), **JWT** (signed stateless claims), **OAuth2** (token delegation w/ scopes/refresh), **Bearer** (token in Authorization header), **mTLS** (mutual TLS certs), **OIDC** (OAuth2+ID tokens), **HMAC** (signed requests), **ISU** (service accounts).[zuplo][apideck] Gateways (Kong/Zuplo) enforce via policies; trade complexity for granularity.[kong]

## Keywords & Patterns
- **Basic Auth**: `Authorization: Basic dXNlcjpwYXNz` (Base64), stateless, every request.[apideck]
- **API Keys**: `X-API-Key: sk_abc123`, query/header, revocable, scoped (read/write).[zuplo]
- **JWT**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` (header.payload.signature), claims (exp/iss/sub/aud/scope).[zuplo]
- **OAuth2**: Flows (auth-code/PKCE/client-creds), scopes, access/refresh tokens, grant types.[apideck]
- **Bearer**: `Authorization: Bearer <token>`, JWT/opaque, stateless validation.[zuplo]
- **mTLS**: Client/server certs, PKI, CRL/OCSP revocation, no tokens over wire.[apideck]
- **OIDC**: `openid` scope, ID tokens (JWT user info), discovery endpoints.[apideck]
- **HMAC**: Sign(request+timestamp+nonce+key), replay protection.[apideck]

## Common Trade-offs + Questions
API Keys: dev-friendly but key exposure = full access; JWT: stateless/fast but no revocation (blacklists bloat); OAuth2: secure delegation but 3-5x complexity; mTLS: zero-trust but PKI overhead.[zuplo] Basic: simple/legacy vs NEVER public; Bearer scales but token theft = compromise.[apideck]

**Example questions**:
- Public API devx vs security? (API Keys + rate limits).[zuplo]
- Microservices auth? (JWT short expiry + refresh).[zuplo]
- 3rd-party user data? (OAuth2 PKCE).[apideck]
- Zero-trust internal? (mTLS + SPIFFE).[1]

| Method | Security | Complexity | Scalability | Revocation |
|--------|----------|------------|-------------|------------|
| Basic | Low | Low | Medium | Password reset [apideck] |
| API Key | Medium | Low | High | Instant block [zuplo] |
| JWT | High | Medium | High (stateless) | Blacklist/jti [zuplo] |
| OAuth2 | Very High | High | High | Token expiry/revoke [apideck] |
| mTLS | Highest | High | High | CRL/OCSP [apideck] |

## Use Cases
- **Public APIs**: API Keys (Stripe/Twilio dev portals).[zuplo]
- **Microservices**: JWT (internal, short-lived).[zuplo]
- **3rd-party**: OAuth2 (Google/Facebook login).[apideck]
- **B2B Enterprise**: mTLS + OAuth (Salesforce/Workday).[apideck]
- **Webhooks**: HMAC signatures (GitHub/Slack).[apideck]
- **SSO**: OIDC (Okta/Auth0).[apideck]

## Big Tech Blogs & Efficiency Wins
Netflix uses JWT + gateway for microservices (stateless, 10x less DB hits). Stripe API keys optimize devx (scoped, instant revoke) for 1M+ devs.[zuplo] Google Cloud mTLS enforces zero-trust, cutting lateral movement 90%. Kong blogs detail hybrid (JWT+Oauth) reducing auth latency 70% via caching.[kong][2][1]

## Cheat-Sheet Q&A
**Q: JWT revocation?** A: Short expiry (15m) + refresh + jti blacklist.[zuplo]  
**Q: OAuth2 PKCE why?** A: Public clients (SPA/mobile) no client_secret.[apideck]  
**Q: mTLS vs JWT?** A: mTLS no bearer tokens over wire; mutual auth.[1]
**Q: API Key security?** A: HTTPS + rotation + scoped + rate limit.[zuplo]  
**Q: Basic Auth ever?** A: Internal/HTTPS-only dev tools.[apideck]  
**Q: OIDC vs OAuth2?** A: OIDC adds ID token (user info); OAuth2 authorization only.[apideck]

## Data Structures & Algorithms
JWT parsing: Base64 decode + HMAC/RS256 verify (hash maps for claims). OAuth token store: Redis (sliding window expiry), Bloom filters (blacklist check). mTLS: X.509 cert chains (tree validation), CRL (distributed hash tables). Rate limiting: Token bucket/Leaky bucket (Redis counters). Scope checks: RBAC graphs (adjacency lists).[zuplo]

## Key Highlights & Tradeoffs
- **DevX**: API Keys > JWT > OAuth2 > mTLS.[zuplo]
- **Security**: mTLS > OAuth2+JWT > API Keys > Basic.[apideck]
- **Scale**: Stateless (JWT/Bearer) > Stateful sessions.[zuplo]
- **Interviews**: "Hybrid: API Gateway terminates auth, passes user context".[kong]
